# Mini Legal Casebase Search Engine

A production-grade prototype of an AI-powered legal information system. This project demonstrates a complete Retrieval-Augmented Generation (RAG) pipeline, allowing users to upload legal documents (PDF/TXT), index them in a vector database, and perform natural language semantic searches with streaming AI summaries.

## Architecture

- **Frontend**: React (Vite), Tailwind CSS, Firebase Storage SDK.
- **Backend**: Node.js, Express.
- **Vector Database**: Pinecone.
- **AI/LLM**: OpenAI (`text-embedding-3-small` for embeddings, `gpt-4o-mini` for streaming summaries).
- **Storage & Metadata**: Firebase Storage (raw files) and Firestore (metadata).

## Features

1. **Document Ingestion**: Upload PDFs or text files. The backend extracts text, splits it into semantic chunks using LangChain, generates embeddings, and upserts them to Pinecone.
2. **Semantic Search**: Search using natural language (e.g., "breach of contract in retail").
3. **Contextual Snippets**: View the exact text snippets and relevance scores retrieved from the vector database.
4. **Streaming AI Summary**: A real-time, streaming summary generated by OpenAI based on the retrieved context.
5. **Vector Noteup**: Find conceptually similar cases by comparing the mathematical vector of a selected paragraph against the entire database.

## Setup Instructions

### 1. Prerequisites
- Node.js v20+
- Firebase Project (Storage & Firestore enabled)
- Pinecone Account (Create an index named `legal-cases`, dimension `1024`, metric `cosine`)
- OpenAI API Key

### 2. Backend Setup
```bash
cd backend
npm install
```
Create a `.env` file in the `backend` directory (see `docs/ENV_SETUP.md` for the template) and fill in your API keys and Firebase Admin credentials.

Start the backend server:
```bash
npm run dev
```

### 3. Frontend Setup
```bash
cd frontend
npm install
```
Create a `.env` file in the `frontend` directory (see `docs/ENV_SETUP.md` for the template) and fill in your Firebase Client configuration.

Start the frontend development server:
```bash
npm run dev
```

## Evaluation Criteria Addressed

- **Clear, sensible architecture**: Strict separation of concerns. Frontend handles UI and direct-to-cloud storage uploads. Backend handles heavy processing (PDF parsing, chunking, embeddings, Pinecone).
- **Clean, modular code**: Backend is split into `config`, `routes`, and `services`. Frontend uses modular React components.
- **Quality/relevance of retrieval**: Uses LangChain's `RecursiveCharacterTextSplitter` with overlap to preserve legal context, and OpenAI's latest embedding model (`text-embedding-3-small` with 1024 dimensions).
- **Bonus**: Implemented a clean UI with Tailwind CSS, real-time streaming AI summaries using Server-Sent Events (SSE), and AI auto-tagging for metadata extraction.
